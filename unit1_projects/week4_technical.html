JavaScript For The Win!

This weeks article discuss's the enumerable method, .cycle. Cycle allows for an argument to be passed to it that deterimines how many times it will iterate through the collection that you call it upon. This allows for a much cleaner way to loop. For example, say you have an array of integers and you want to find out the sum of all integers stored. You could do this with:
array = [1, 2, 3, 4]

num = array.length
sum = 0
x = 0
while num != 0 do
    sum = sum + array[x]
    x += 1
    num -= 1
end
puts sum

# output is 10!

This is long and tedious, and while their are other solutions, the cycle option is much more efficient here:

array = [1, 2, 3, 4]

sum=0
array.cycle(1) {|x| sum = sum + x}
puts sum

# output is 10!

Much cleaner! Less code, less variables used, less to keep track of. Keep in mind that the argument that you are passing to .cycle is not directing what arguments you are cycling to, but how many times you are cycling through the entire collection. Watch ouf for that endless loop, make sure you tell .cycle how many times you want it to run!
