Understanding Class

This week's article discusses the useful and powerful Ruby Class. When I first started learning to code with Ruby, the Class was a dark and mysterious unknown to me. I had no concept of, a.) what the purpose of it was, or b.) why anyone would take the extra work to create one when they could just write out the code and move on. Keep in mind that the complexity of what I was doing was naming vairables and adding sums. I really had no real world experience or knew how to even begin to create usefull code. This week has changed everything for me. The use of Class to create your own new object with variables and methods greatly simplifies and organizes your code. Let's look at an example of the class works.

My daughter's soccer team won last week, and I have decided that I am going to write a quick little program to keep track of her team and the rest of the league. I'm going to need a way to store and modify this infomration easily, so I'm going to create a new class to assist in this. I'm going to call this class League (notice the capitalizaion on your new class name? Yah you need it!) Inside your class you will need a constructer method which will set the baselines for your new object. In Ruby, this is the initialize method, and should be placed near the beginning of your class. For this class, I am going to need use a hash to store different team names, and their relative wins and losses. So, in my initialize constructer, I'm going to give the new League an empty hash to work with.

Great! Now my league has something to work with. So what else will I need here? I will need the ability to add teams to the league, and to assign wins and losses throughout the season. This will require me to design methods to do such. Lets back up a second and look at that last piece of code. Did you notice the @ in front of the variable name for the hash? This is designating that variable as an instance variable. This variable will be available anywhere withen this instance of the new league, but not outside. The same pertains to any method that you create inside your class. The method will only be availble to be called upon by the class it self, and not some other oustide force.
